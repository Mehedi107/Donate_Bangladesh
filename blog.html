Important FAQ

What is the Document object model (DOM)?

The Document Object Model (DOM) is like a map of a web page that organizes everything—like text, images, and links—into objects that can be easily changed with programming. Imagine the web page as a tree, where each part (such as a header or paragraph) is a branch. The DOM lets developers grab any part of this tree and change it, move it, or add new parts while the page is still open. This makes it possible to update a web page without reloading it, allowing for more interactive and dynamic user experiences.

How do you select an element form the DOM?

To select an element from the DOM in JavaScript, there are several methods. The most common ones are:
1. getElementById()
2. getElementsByClassName()
3. getElementsByTagName()
4. querySelector()
5. querySelectorAll()

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM where instead of attaching event listeners to individual child elements, you attach a single event listener to a parent element. This parent element listens for events that occur on its child elements, leveraging the way events "bubble up" through the DOM.
